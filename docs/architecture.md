# Архитектура и Структура проекта

## Структура директорий
```
src/
├── api/            # Модули для взаимодействия с внешними API
│   ├── auth.ts     # API авторизации
│   ├── axios.ts    # Настроенный экземпляр axios
│   ├── github.ts   # API GitHub (получение профиля и репозиториев)
│   ├── orders.ts   # API заказов
│   └── reviews.ts  # API отзывов
├── assets/         # Статические ресурсы (шрифты, изображения)
├── components/     # Переиспользуемые UI компоненты
│   ├── Layout/     # Основной макет страницы (Header, Footer, Main)
│   ├── Navbar/     # Навигационная панель (включая мобильное меню)
│   └── ...
├── context/        # React Context (глобальное состояние)
│   └── AuthContext.tsx # Контекст авторизации
├── pages/          # Компоненты страниц (маршруты)
│   ├── Home/       # Главная страница
│   ├── Contacts/   # Страница контактов
│   ├── Login/      # Страница входа
│   ├── Order/      # Страница оформления заказа
│   ├── Reviews/    # Страница отзывов
│   └── ...
├── types/          # TypeScript определения типов
│   └── index.ts    # Общие интерфейсы (User, Repo, Order и т.д.)
├── App.tsx         # Корневой компонент с настройкой маршрутизации
└── main.tsx        # Точка входа приложения
```

## Ключевые компоненты

### Layout
Обертка для всех страниц приложения. Содержит `Navbar` и `Footer` (если есть), а также `CookieConsent`. Обеспечивает единую структуру страниц.

### Navbar
Отвечает за навигацию.
*   **Desktop:** Горизонтальное меню ссылок.
*   **Mobile:** Гамбургер-меню, реализованное через `Drawer` (Material UI).
*   **Особенности:** Использует `useLocation` для подсветки активного пункта меню, чтобы избежать конфликтов типов между `react-router-dom` и MUI.

### Home Page
Главная страница, агрегирующая информацию.
*   Загружает данные профиля и репозиториев через `githubApi`.
*   Фильтрует и отображает конкретные проекты (`visuliser-backend`, `pd-projects-backend` и др.) с кастомными описаниями.

## Взаимодействие с API
Все запросы к API вынесены в папку `src/api/`.
*   **GitHub API:** Используется для получения публичных данных (без авторизации или с токеном, если настроено).
*   **Backend API:** Взаимодействие с собственным бэкендом (авторизация, отправка форм). Используется `axios` с интерцепторами для обработки токенов.

## Стилизация
*   **CSS Modules / CSS файлы:** Основной способ стилизации компонентов (например, `Home.css`, `Navbar.css`).
*   **Material UI:** Используется для сложных интерактивных компонентов (иконки, Drawer, Inputs) и базовой сетки в некоторых местах.
*   **Адаптивность:** Реализована через стандартные CSS Media Queries. Основные точки перелома: `768px` (планшеты) и `480px` (мобильные телефоны).

## Типизация
Проект полностью написан на TypeScript. Все основные сущности (`GitHubUser`, `GitHubRepo`, `Order`) описаны в `src/types/index.ts`. Это обеспечивает безопасность типов при передаче данных между компонентами и API.
