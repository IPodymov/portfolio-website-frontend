# Деплоймент (Развертывание)

## Сборка приложения
Проект собирается с помощью Vite. Конфигурация сборки находится в `vite.config.ts`.

Для выполнения сборки выполните команду:
```bash
npm run build
```

Эта команда выполнит следующие действия:
1.  **tsc -b**: Проверка типов TypeScript.
2.  **vite build**: Сборка и оптимизация ассетов.

Результат сборки будет помещен в директорию `dist/`. Эта папка содержит все необходимые статические файлы (HTML, CSS, JS, шрифты) для развертывания на любом веб-сервере.

## Варианты хостинга

### Static Hosting (Netlify, Vercel, GitHub Pages)
Так как это SPA (Single Page Application), его можно легко развернуть на статических хостингах.

**Важно для SPA:**
При настройке сервера необходимо убедиться, что все запросы, не соответствующие статическим файлам, перенаправляются на `index.html`. Это нужно для корректной работы клиентского роутинга (`react-router-dom`).

*   **Netlify:** Обычно определяет настройки автоматически. Если нет, создайте файл `_redirects` в папке `public/` с содержимым: `/* /index.html 200`.
*   **Vercel:** Работает из коробки.
*   **Nginx:** Пример конфигурации location:
    ```nginx
    location / {
      try_files $uri $uri/ /index.html;
    }
    ```

## Переменные окружения
Если приложение использует переменные окружения (например, API URL), убедитесь, что они настроены на сервере сборки или в панели управления хостинга.
В Vite переменные окружения должны начинаться с префикса `VITE_`.

Пример `.env`:
```
VITE_API_URL=https://api.example.com
```
